Why MiaoWallet?

❌ The Problem

AI agents need blockchain access, but current approaches are broken:

  • Give agent the private key → total loss if compromised
  • Manual approval per tx → defeats automation
  • Shared .env files → keys in plaintext, easily leaked
  • No audit trail → can't verify what AI did

✅ The Solution

MiaoWallet separates key custody from transaction logic:

  • Keys in macOS Keychain — never leave the device
  • Session-based signing — time & count limited
  • Whitelist control — approved DApps only
  • Walrus attestation — every action auditable

Security Model

🔐 Key Isolation

Private keys in macOS Keychain (hardware-backed). Never in files, env vars, or agent memory. Each access requires user authorization.

⏱ Session Limits

Time-limited + count-limited signing sessions. Auto-expire, no permanent keys. Instant manual revocation.

🛡 Whitelist Control

Origin whitelist (approved DApp domains) + Contract whitelist (approved Move packages). Unauthorized = rejected.

📋 Walrus Audit Trail

Every transaction stored as immutable attestation on Walrus decentralized storage. AI behavior fully auditable.

🔒 Keeper Encryption

Automated bots: PBKDF2 + Fernet encrypted keys. Decrypted only in memory at startup. .env never has plaintext.

🚫 Revocation

One-click session kill in WebGUI. Agent immediately loses signing ability. No residual access.
