Why MiaoWallet?

❌ The Problem

AI agents need blockchain access, but existing solutions either expose private keys to the agent (unsafe) or require manual approval for every transaction (slow). There's no secure, automated way for agents to operate on-chain.

✅ The Solution

MiaoWallet separates key custody from transaction logic. The AI agent decides what to do; MiaoWallet handles the signing — securely, automatically, within user-defined limits. Keys never leave the device.

Security Model

🔐 Key Isolation

Private keys stored in macOS Keychain (secure enclave). Never in files, env vars, or agent-accessible memory.

⏱ Session Limits

Time-limited, count-limited signing sessions. Access auto-expires. No permanent API keys.

🛡 Bridge Architecture

Agent only receives signatures, never touches private keys. Even if compromised, keys are safe.

📋 Audit Trail

Every operation stored as immutable attestation on Walrus decentralized storage.

✅ Whitelist Control

Restrict which DApp origins and contracts the agent can interact with. Prevent unauthorized calls.

Origin & Contract whitelist — only approved DApps can request signatures
